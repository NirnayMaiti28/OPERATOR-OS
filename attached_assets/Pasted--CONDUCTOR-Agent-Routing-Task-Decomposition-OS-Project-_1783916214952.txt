# CONDUCTOR: Agent Routing & Task Decomposition OS

## Project Specification

Build a production-grade multi-agent orchestration system that intelligently decomposes complex user queries into subtasks and routes them to specialized agents. System must outperform or match AutoGen/LangChain on latency, cost, and task success rate.

## Tech Stack
- **Backend**: Python 3.11+ with FastAPI
- **LLM**: Groq API (Claude 3.5 Sonnet - free tier for decomposition & reasoning)
- **Storage**: SQLite for task logs, Redis optional for state
- **Agent Execution**: Async/concurrent with timeout handling + public API integration
- **Public APIs**: Weather, News, Wikipedia, GitHub, Stock Markets, OpenExchangeRates
- **Benchmarking**: pytest + custom evaluation harness

## Core Architecture

### 1. Task Decomposer
- Input: User query (string)
- Process: Use Groq Claude 3.5 Sonnet to break query into atomic subtasks with:
  - Task description
  - Required capabilities (retrieval, code, reasoning, data-analysis, tool-integration)
  - Dependencies (task A must complete before B)
  - Estimated tokens/cost (Groq free tier tracking)
  - Tool hints (if this looks like a weather/news/finance query, flag for ToolAgent)
- Output: Task graph (DAG) with metadata
- Cost: Free (Groq free tier)

### 2. Agent Router
- Registry of available agents with capability tags and performance history
- Routing logic:
  - Match task requirements to agent capabilities
  - Use latency/cost history to select best agent for task type
  - Fallback to secondary agent if primary fails
  - No hardcoded routing — fully dynamic
- Output: Agent assignment with retry strategy

### 3. Agent Abstraction Layer
- Base `Agent` class with:
  - `execute(task: Task) → Result`
  - `capabilities: List[str]`
  - `performance_stats: Dict`
  - `cost: float` (track API costs)
- Concrete agents (minimum 5):
  - **RetrievalAgent**: BM25 + dense embedding search (SBERT/ColBERT)
  - **ReasoningAgent**: Groq Claude for chain-of-thought + verification
  - **CodeAgent**: Safe code execution with timeout + sandboxing
  - **DataAnalysisAgent**: Pandas/polars operations (local)
  - **ToolAgent**: Orchestrates public APIs:
    - Weather API (OpenWeatherMap or WeatherAPI)
    - News API (NewsAPI for trending)
    - Wikipedia API (research/factual lookup)
    - GitHub API (code search/repo info)
    - Stock API (Alpha Vantage or similar)
    - Currency Exchange API (OpenExchangeRates)
  - ToolAgent has internal router: maps task type → correct public API → format response
- Each agent logs execution time, token usage, API cost, success/failure

### 4. Orchestrator
- Execute task graph with:
  - Parallel execution where dependencies allow
  - Task-level timeout (configurable, default 30s)
  - Graceful failure handling (fallback agents, partial results)
  - State persistence (checkpoint every task completion)
- Return: Aggregated result + execution trace

### 5. Observability
- Structured logging (task ID, agent ID, latency, tokens, status)
- SQLite schema for audit trail
- CLI/API endpoint to query execution history

## Requirements

### Functional
1. Decompose queries without hallucinating tasks
2. Route tasks to correct agent within 500ms (excluding LLM latency)
3. Execute 3+ concurrent tasks in parallel
4. Recover from single agent failure (try fallback or alternate API)
5. Auto-detect public API needs and route to ToolAgent + correct API
6. Handle API rate limits gracefully (queue, retry, degrade)
7. Fall back to local reasoning if API fails
8. Log all executions + API calls for audit

### Non-Functional
1. Latency: Total time < 2x native single-LLM call for comparable tasks
2. Cost: Free tier sustainable (Groq free + free API tier usage)
3. Reliability: 95%+ task success rate on benchmark (including API failures)
4. Code quality: Type-hinted, PEP-8, no bare excepts, docstrings on all functions
5. API resilience: 90%+ success rate even with 1-2 APIs down

## Configuration & Setup

### Required API Keys (.env file)
```
GROQ_API_KEY=<free from groq.com>
OPENWEATHER_API_KEY=<free from openweathermap.org>
NEWSAPI_API_KEY=<free from newsapi.org>
GITHUB_TOKEN=<free from github.com/settings/tokens>
ALPHA_VANTAGE_KEY=<free from alphavantage.co>
EXCHANGERATE_API_KEY=<free from exchangerate-api.com>
```

### Installation
```bash
pip install -r requirements.txt
export $(cat .env | xargs)
python -m pytest tests/  # run tests
python -m benchmarks.evaluate  # run benchmark suite
uvicorn src.conductor.api:app --reload  # run API
```
